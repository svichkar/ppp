 Вариант 1.

Создать класс, который производит последовательно сериализацию и архивирование объектов.
 Один метод должен получать объект в качестве параметра и возвращать массив байт, представляющих собой заархивированный объект.
 Второй метод должен выполнять обратную операцию.
Методы должны быть generic.

Вариант 2.

Создать свой класс, который будет представлять сущность Account. Класс должен содержать следующие поля: 
id: Long
name: String
role: String
Класс должен быть сериализируемым. 
Сериализация полей должна представлять собой конкатенацию значений каждого поля в строку и после этого запись итоговой строки в файл с расширением .dat.
