Библиотека
Необходимо разработать приложение для учета книг в библиотеке. 
В приложении должны быть реализованы следующие возможности:
1. Учет книг.
2. Учет категорий книг.
3. Учет авторов книг.
4. Учет ячеек для книг.
5. Учет клиентов библиотеки.
6. Учет арендованных книг.
7. Поиск книг по названию/авторам/категориям.
8. Поиск книг в библиотеке.
9. Поиск книг, отданных в аренду.



https://docs.google.com/document/d/1WoX7F1zOpUeAZzDkTpFDwO4xq1WAH2w_q6CqL84hwB4/edit

ОБЯЗАТЕЛЬНЫЕ ЗАДАНИЯ:

7. Написать несколько выборок со вложенными запросами в условиях.
8. Написать запрос с использованием ORDER BY
9. Написать запрос с использованием  UNION (можно делать из одной таблицы двумя запросами с разными условиями)
10. Написать 2 или более запроса с использованием INNER JOIN. Используйте алиасы для именования таблиц.
11. Написать 2 или более запроса с использованием LEFT(RIGHT) OUTER JOIN. Используйте алиасы для именования таблиц.
12. Написать один или более запросов с использованием числовых функций
13. Написать один или более запросов с использованием строковых функций
14. Написать один или более запросов с использованием календарных функций
15. Написать два или более запросов с использованием агрегатных функций. Хотя бы один запрос должен использовать GROUP BY и HAVING.

16. Все запросы необходимо поместить в файл select.sql и залить в вашу рабочую папку с проектом.


ТРЕБОВАНИЯ:

1. Все ключевые слова SQL должны быть в верхнем регистре, а наименование таблиц, столбцов... - должны быть в нижнем регистре. Пример: SELECT first_name, last_name FROM user WHERE first_name = ‘John’ AND age < 18
2. Разделять команды необходимо с помощью символа “;”.
3. Все скрипты должны быть рабочими.
4. Все выборки должны выдавать результаты, после выполнения ваших ddl.sql и dml.sql.
5. Очень желательно чтобы выборки имели смысловую нагрузку и присутствовали комментарии к более сложным выборкам. Комментарии начинаются с “--”
