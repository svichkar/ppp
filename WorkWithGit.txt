Мы будем работать с репозиторием на bitbucket 








(https://bitbucket.org/nixsolutions/javappp/overview). Для этого необходимо:
Зарегистрироваться на сервисе
Установить себе git: https://help.github.com/articles/set-up-git/
Изучить инструкцию на той же ссылке
Убедиться, что пункт 2 и 3 инструкции выполнены и имя / e-mail прописаны в git --global

Работаем мы с репозиторием, находящемся на bitbucket. В репозитории по адресу https://bitbucket.org/nixsolutions/javappp/overview нужно скопировать строку адреса репозитория (git@bitbucket.org:nixsolutions/javappp.git) и склонировать его себе в папку, где будут выполняться работы:
cd path/to/you/folder
git clone git@bitbucket.org:nixsolutions/javappp.git

В репозитории для каждого из Вас будет создана ветка, с которой Вы будете работать. В работе вам будут необходимы такие команды git как:
клонирование репозитория
переключение с ветки на ветку
получение модификация с удаленного репозитория в локальную копия
коммит ваших изменений из рабочей папки в локальный репозиторий
отправка модификаций из локального репозитория в удаленный
возможно, объединение локальных изменений с изменениями в удаленном репозитории

Для этого вам понадобятся такие команды git:
git clone git@bitbucket.org:nixsolutions/javappp.git
git checkout [ваша ветка]
git pull --rebase
git add -A (добавление всех новых файлов в индекс); 
git commit -a -m [ваш коментарий к коммиту] 
git push
git merge [ветка из которой надо забрать изменения в текущую ветку]

Перед началом работы следует ознакомиться с документацией по этим командам тут:
https://git-scm.com/documentation (английская)
http://git-scm.com/book/ru/v1 (русская)


Примерный план работы с репозиторием
Перехожу в свою рабочую папку, куда склонирован репозиторий
Создаю свою ветку: git checkout -b tsimbaluk
Работаю, создаю папку и файл java/feature.java
Добавляю все новые файлы в индекс: git add .
Делаю коммит: git commit -m "first commit"
Делаю push: git push -u origin tsimbaluk
Так как ветка таким push была добавлена на удаленный репозиторий, в последующем можно использовать просто git push, а если ушли с ветки и надо снова на нее вернуться -  git checkout tsimbaluk
Если есть изменения на удаленном репозитории и надо забрать их себе делаю git pull --rebase
